# Improvement Suggestions

**Prepared by:** Ruknuddin Asrari  
**Role:** Software Quality Engineer  
**Organization:** The Career Insights Hub LLC

## Recommended Improvements

1. Increase text readability by using readable font sizes, proper spacing, and clear section headings.
2. Add descriptive alt text for all meaningful images.
3. Use empty alt text for decorative images so screen readers can skip them.
4. Add strong visible focus indicators for all buttons, links, inputs, menus, and controls.
5. Add a skip-to-content link for keyboard users.
6. Improve color contrast for light gray text and disabled/secondary UI states.
7. Do not rely only on color to identify links or important UI states.
8. Ensure search inputs and form fields have visible labels or accessible aria-labels.
9. Use aria-live or role=alert for validation and error messages.
10. Maintain a logical heading structure using one H1 and sequential H2/H3 headings.
11. Verify all controls can be reached and operated using only the keyboard.
12. Retest accessibility after fixes using manual review and browser accessibility tools.

## Retest Recommendation

After fixes are implemented, perform a regression accessibility test covering keyboard navigation, screen-reader label checks, color contrast validation, and form error handling.
